# ExpoNotes Backend

Сервис канбан-досок (клон Trello) на FastAPI.  
Учебный pet-проект для отработки асинхронной работы с PostgreSQL, интеграции с S3-хранилищем (MinIO), JWT-авторизации, оптимистичных блокировок и real‑time обновлений через WebSocket.  
Бэкенд и фронтенд полностью функционалены.

## Основные возможности

**Пользователи и авторизация**
- Регистрация и вход (JWT access + refresh токены).
- Обновление токенов.
- Выход с аннулированием токенов (blacklist).
- Текущий профиль, обновление данных (username, email, bio, дата рождения, пол, аватар).
- Поиск пользователя по email или ID.
- Список пользователей с курсорной пагинацией.

**Доски**
- Создание доски (создатель становится владельцем с ролью `owner`).
- Получение списка досок, доступных пользователю (как участник или владелец).
- Просмотр, обновление (с версионированием) и удаление доски.
- Управление участниками: добавление, изменение роли (`reader`, `writer`, `owner`), удаление.
- Только владелец может изменять состав участников и удалять доску.

**Колонки и карточки**
- Создание, просмотр, обновление (с версионированием) и удаление колонок в рамках доски.
- Создание, просмотр, обновление, удаление карточек в колонках.
- Перемещение карточек между колонками с изменением порядка.
- Поля карточки: название, описание, исполнитель, срок, приоритет (`low`/`medium`/`high`), файл.

**Комментарии**
- Добавление, просмотр, редактирование (только автор) и удаление (автор или владелец доски) комментариев к карточкам.

**Файлы**
- Загрузка в S3-совместимое хранилище MinIO.
- Скачивание целиком или частями (поддержка HTTP Range).
- Получение метаданных и удаление файла.

**Поиск**
- Поиск карточек по тексту (название, описание) во всех досках, доступных пользователю.
- Курсорная пагинация результатов.

**Аудит**
- Журнал действий пользователей с фильтрацией по типу сущности и ID.
- Курсорная пагинация логов.

**Real‑time обновления (WebSocket)**
- Подписка на обновления конкретной доски.
- Автоматическая рассылка событий (например, перемещение карточки) всем подписчикам доски.
- Аутентификация через JWT-токен в параметрах соединения.

**Документация OpenAPI**
- Полное описание всех эндпоинтов доступно после запуска по адресу `/docs` (Swagger UI).

## Схема базы данных

### users
| Поле            | Тип           | Описание                        |
|-----------------|---------------|----------------------------------|
| id              | int (PK)      | Уникальный идентификатор         |
| username        | varchar       | Имя пользователя                 |
| email           | varchar       | Уникальный email                 |
| hashed_password | varchar       | Хэш пароля                       |
| avatar_id       | int (FK?)     | ID загруженного аватара          |
| bio             | text          | Краткая информация               |
| birth_date      | date          | Дата рождения                    |
| gender          | varchar       | "male" / "female"                |
| created_at      | timestamptz   | Дата регистрации                 |

### refresh_tokens
| Поле       | Тип          | Описание                         |
|------------|--------------|----------------------------------|
| id         | int (PK)     | Идентификатор токена             |
| user_id    | int (FK)     | Внешний ключ к users.id          |
| token      | varchar      | Уникальный refresh-токен         |
| expires_at | timestamptz  | Срок действия токена             |
| created_at | timestamptz  | Дата создания                    |

### blacklisted_tokens

| expires_at | timestamptz  | Срок истечения токена            |

### files

| content_type  | varchar      | MIME‑тип                         |

| uploaded_by   | int (FK)     | Внешний ключ к users.id          |

### boards

| description | text         | Описание                         |
| order       | int          | Порядок в колонке                |
| author_id   | int (FK)     | Автор карточки (users.id)        |
| assignee_id | int (FK)     | Исполнитель (users.id)           |
| due_date    | timestamptz  | Срок выполнения                  |
| priority    | enum         | Приоритет: low, medium, high     |
| file_id     | int (FK)     | Прикреплённый файл (files.id)    |
| version     | int          | Версия для оптимистичной блокировки|
| created_at  | timestamptz  | Дата создания                    |

### comments
| Поле       | Тип          | Описание                         |
|------------|--------------|----------------------------------|
| id         | int (PK)     | Идентификатор комментария        |
| card_id    | int (FK)     | Внешний ключ к cards.id          |
| author_id  | int (FK)     | Автор комментария (users.id)     |
| text       | text         | Текст комментария                |
| created_at | timestamptz  | Дата создания                    |

### audit_log
| Поле        | Тип          | Описание                         |
|-------------|--------------|----------------------------------|
| id          | int (PK)     | Идентификатор записи             |
| user_id     | int (FK)     | Пользователь, выполнивший действие|
| action      | enum         | Тип действия (create, read, update, delete, move, assign) |
| entity_type | enum         | Тип сущности (board, column, card, comment, file, board_member) |
| entity_id   | int          | ID изменённой сущности           |
| changes     | jsonb        | Детали изменений (опционально)   |
| created_at  | timestamptz  | Время события                    |

Все связи обеспечиваются внешними ключами. Каскадное удаление настроено: при удалении доски удаляются колонки, карточки, комментарии и участники; при удалении колонки – её карточки и т.д.

## Быстрый запуск (Docker Compose)

1. Убедитесь, что установлены Docker и Docker Compose.

2. Склонируйте репозиторий

3. Настройте переменные окружения.  
   В корне проекта лежит `.env.example` — скопируйте его в `.env`, раскомментируйте и при необходимости измените параметры (учётные данные БД, MinIO, секретный ключ JWT).
4. Запустите проект:
   ```bash
   docker compose up -d --build
   ```
5. После запуска:
   - Backend API доступен на `http://localhost:3001`
   - Swagger UI с полным описанием методов: `http://localhost:3001/docs`
   - Web‑консоль MinIO: `http://localhost:9001` (логин/пароль из `MINIO_ROOT_USER`/`MINIO_ROOT_PASSWORD`)
   - База данных и S3‑бакет создаются автоматически.
   - Фронтенд необходимо запустить, открыв файл index.html в браузере любым способом

**Важно:** Перемещение карточек работает в реал-тайме через вебсокет соединение. Чтобы увидеть это рекомендую открыть 2 фронтенда: один в обычном бразузере, а второй в другом или в режиме инкогнито. Переместив карточку в одном фронтенде, вы заметите как данные обновились и на другом без обновления страницы

## Стек технологий (backend)

- **FastAPI** (Python) — асинхронный веб-фреймворк
- **SQLAlchemy** (asyncpg) + **PostgreSQL 17** — работа с базой данных
- **MinIO** (aiobotocore) — S3-совместимое хранилище файлов
- **JWT-авторизация** (python-jose, passlib) — access и refresh токены
- **WebSocket** — real‑time уведомления о перемещении карточек
- **Docker Compose** — развёртывание всех сервисов
- **Frontend** - ванильный html/css/js
